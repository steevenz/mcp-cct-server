# Domain: Learning

## Purpose
The Learning domain establishes the autonomous evolutionary cycle of the CCT server. It turns historical session data into durable architectural wisdom (Thinking Patterns) and defensive heuristics (Anti-Patterns).

## Scope
Included in this domain:
- **Instinct Extraction**: Atomic learning of tool call patterns and user preferences.
- **Pattern Evolution**: Promoting project-scoped instincts to global standards.
- **Project Scoping**: Isolating project-specific conventions to prevent cross-contamination.

## Usage
The `POST_MISSION_LEARNING` strategy is executed as the final phase of every major cognitive pipeline. It ensures that the system grows smarter with every interaction.
