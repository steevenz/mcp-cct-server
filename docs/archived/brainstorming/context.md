# Domain: Brainstorming

## Purpose
The Brainstorming domain governs the discovery phase of a mission. It enforces the **Socratic Gate**, ensuring that no implementation begins without a clear understanding of purpose, users, and scope.

## Scope
Included in this domain:
- **Socratic Gate**: Mandatory 3 questions before implementation for vague requests.
- **Dynamic Questioning**: Generating architectural decision points with trade-offs.
- **Mission Transparency**: Visual status reporting (✅🔄⏳❌).

Excluded from this domain:
- **Planning Patterns**: Handled by the `Planning` domain.
- **Engineering SOP**: Handled by the `Engineering` domain.

## Usage
Activate the `BRAINSTORMING` strategy at the start of any `FEAT`, `ARCH`, or `ENGINEERING` track where requirements are not 100% deterministic.
