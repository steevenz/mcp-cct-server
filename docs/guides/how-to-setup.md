# Guide: How to Setup CCT MCP Server

This guide provides step-by-step instructions to get the **Creative Critical Thinking (CCT)** server running and integrated into your development workflow.

## 1. Prerequisites
- **Python 3.10 or higher** installed.
- **Git** installed.
- (Recommended) An API key for **Google Gemini 1.5 Flash** (via [Google AI Studio](https://aistudio.google.com/)).

## 2. Installation

```bash
# Clone the repository
git clone https://github.com/steevenz/mcp-cct-server.git
cd mcp-cct-server

# Create a virtual environment
python -m venv .venv

# Activate the venv
# Windows:
.\.venv\Scripts\activate
# macOS/Linux:
# source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

## 3. Configuration

### Environment Variables (.env)
Copy the example environment file and edit it:

```bash
cp .env.example .env
```

Open `.env` and configure your preferences:

| Key | Value |
|-----|-------|
| `CCT_LLM_PROVIDER` | `gemini`, `openai`, `anthropic`, or `ollama` |
| `GEMINI_API_KEY` | Your Google API Key |
| `CCT_DEFAULT_MODEL` | The model name (e.g., `gemini-1.5-flash`) |

> [!TIP]
> **Guided Fallback**: If you leave `CCT_LLM_PROVIDER` empty, the server will operate in **Guided Mode**, providing structured instructions instead of autonomous reasoning.

## 4. IDE Integration

### Windsurf / Verdent.ai
Add the following to your `mcpServers` configuration (usually in `C:\Users\<YOU>\.verdent\mcp_config.json` or similar):

```json
{
  "mcpServers": {
    "cct-cognitive-server": {
      "command": "C:/PATH/TO/mcp-cct-server/.venv/Scripts/python.exe",
      "args": [
        "-u",
        "C:/PATH/TO/mcp-cct-server/src/main.py"
      ],
      "env": {
        "PYTHONIOENCODING": "utf-8",
        "PYTHONUNBUFFERED": "1",
        "CCT_TRANSPORT": "stdio",
        "CCT_LOG_LEVEL": "INFO",
        "CCT_LLM_PROVIDER": "gemini",
        "GEMINI_API_KEY": "YOUR_KEY_HERE"
      }
    }
  }
}
```

### Claude Desktop
Add similar configuration to your `claude_desktop_config.json`.

## 5. Troubleshooting (The "Hang" Issue)

If the server appears to "hang" or stay silent when using the `thinking` tool:

1.  **Check Paths**: Ensure the `command` and `args` in your JSON config use absolute paths and the correct slashes (`/`).
2.  **Venv Check**: Ensure you are using the `python.exe` inside `.venv/Scripts/` (Windows) or `.venv/bin/` (Linux).
3.  **Logs**: Check the `stderr` logs in your IDE's MCP console. If you see `LLM Call failed`, verify your API keys and internet connection.
4.  **Ollama**: If using `ollama`, ensure the Ollama service is running locally (`ollama serve`).

## 6. Verification
Once connected, try running the `thinking` tool:
- **Autonomous Mode**: You should see a detailed response generated by the server's internal LLM.
- **Guided Mode**: You should see a structured protocol/template for you to follow.

## 7. Windows Background Service (Optional)

You can run the CCT server as a native Windows service. This allows it to run silently in the background, starting automatically with your computer.

### Step 1: Installation
Run the provided PowerShell script as **Administrator**:

```powershell
# In PowerShell (Admin)
cd C:/PATH/TO/mcp-cct-server
./scripts/setup_windows_service.ps1
```

### Step 2: MCP Configuration (SSE)
Since the background service cannot use the `stdio` transport, you must configure your IDE to connect via **SSE (Server-Sent Events)** using the "cool" semantic path.

Update your `mcpServers` config:

```json
{
  "mcpServers": {
    "cct-cognitive-server": {
      "url": "http://localhost:8000/cognitive-api/v1/sync"
    }
  }
}
```

### Step 3: Observability & Widgets
The server now exposes REST endpoints for health monitoring and third-party widgets:

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/health` | `GET` | Quick liveness check. |
| `/status` | `GET` | Detailed telemetry (uptime, LLM provider, active sessions). |

Example `/status` response:
```json
{
  "server": "cct-mcp-server",
  "uptime_seconds": 3600,
  "llm_provider": "gemini",
  "transport": "SSE"
}
```

### Step 4: Managing the Service
- **Stop**: `Stop-Service CCTCognitiveOS`
- **Start**: `Start-Service CCTCognitiveOS`
- **Restart**: `Restart-Service CCTCognitiveOS` (required after editing `.env`)
- **Logs**: Located at `database/logs/cct_service.log`.
