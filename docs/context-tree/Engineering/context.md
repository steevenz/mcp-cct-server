# Domain: Engineering

## Purpose
The Engineering domain defines the standardized workflows for AI-driven technical implementation. It enforces a "Standard Operating Procedure" (SOP) that prioritizes validation before execution and decomposition before implementation.

## Scope
Included in this domain:
- **Engineering Deconstruction**: Methodologies for memetic breakdown of tasks.
- **Eval-First Loops**: Frameworks for defining success criteria before code generation.
- **Model Tier Routing**: Standards for selecting Haiku, Sonnet, or Opus based on complexity.
- **Human Orchestration**: The delegate-review-own model for human-AI collaboration.

Excluded from this domain:
- **Language-Specific Idioms**: General coding style guides (handled by `Guides`).
- **Build Systems**: CI/CD technical details (handled by `Setup`).

## Usage
AI Agents must reference this domain as the **mandatory first step** (`ENGINEERING_DECONSTRUCTION`) for all `FEAT` and `ARCH` pipelines to ensure architectural alignment and risk mitigation.

## C&C Framework Alignment
Engineering represents the **Practical** realization of **Critical** constraints. It ensures that **Creative** features are built on a verified, resilient foundation.
- **Convergence**: `Eval-First` ensures that the implementation converges on the user's success criteria.
- **Immunity**: `Task Decomposition` prevents the "monolithic failure" anti-pattern.
