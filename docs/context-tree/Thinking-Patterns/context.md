# Domain: Thinking-Patterns

## Purpose
The Thinking-Patterns domain serves as the central repository for specialized cognitive strategies, mental models, and structured reasoning archetypes. It defines how the AI adapts its logic to specific problem domains.

## Scope
Included in this domain:
- **Primitives**: Atomic thinking modes (e.g., Linear, Lateral, Dialectical).
- **Heuristics**: Mental shortcuts and high-performance patterns.
- **Cognitive Schemas**: Structured input/output formats for specialized thoughts.
- **Pattern Archiving**: Mechanism for saving and retrieving high-performance reasoning paths.

Excluded from this domain:
- **Execution Orchestration**: Handled by the `CCT` domain.
- **Convergent Synthesis**: Handled by the `Fusion` domain.

## Usage
Reference this domain when:
- Designing new cognitive primitives.
- Investigating the specific "Golden Skills" available to the engine.
- Extending the variety of mental models available for routing.
