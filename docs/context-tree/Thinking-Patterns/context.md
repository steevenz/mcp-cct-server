# Domain: Thinking Patterns

## Purpose
The Thinking Patterns domain is the repository for the **Cognitive DNA** of the CCT server. It implements a **Recursive Learning Loop** by distilling raw session data into high-value knowledge. It serves as both the system's "Peak Wisdom" (Golden Patterns) and its "Mental Immune System" (Anti-Patterns), ensuring that the server grows more intelligent and resilient with every mission.

## Scope
Included in this domain:
- **Golden Thinking Patterns**: Individual documents representing peak reasoning steps (Logic > 0.9).
- **Anti-Patterns**: Forensic records of cognitive failures used to prevent strategy regression.
- **Knowledge Injection**: Mechanisms for bootstrapping new sessions with historical knowledge.
- **Adaptive Thresholds**: Dynamic criteria used to identify elite thoughts or early convergence.

Excluded from this domain:
- **Session History**: Raw, unscored thoughts from active sessions (handled by `Memory`).
- **Engines**: The logic that *generates* patterns (handled by `Engines` and `Primitives`).

## Usage
Developers and AI Agents should reference this domain to:
- **Warm-start** new sessions by injecting relevant Golden Patterns as context.
- **Audit Failures** by reviewing Anti-Patterns to understand why certain strategies failed.
- **Optimize Performance** by leveraging Early Convergence triggers to save tokens.

## C&C Framework Alignment
Thinking Patterns represent the culmination of the **Critical** pillar's regulatory force. They are the thoughts that have been "Naturalized" into the system's long-term memory. 
- **Convergence**: Golden Patterns represent the "Ideal Convergence" for a given problem set.
- **Immunity**: Anti-Patterns represent the "Critical Guardrails" derived from past errors, protecting future **Creative** expansion from repeating known logical pitfalls.
