# Domain: Engines

## Purpose
The Engines domain defines the "Trinity of Cognitive Execution." It contains the heavy-lifting logic modules that orchestrate depth (Sequential), synthesis (Fusion), and persistence (Memory) across every cognitive mission.

## The Trinity (Sub-Domains)

### [🔄 Sequential Control](./sequential/context.md)
Govern the depth and convergence of a session. It manages thought numbering, adaptive limits, and the "Is-Thinking" state.

### [⚛️ Fusion Orchestration](./fusion/context.md)
The synthesis engine that bridges multiple strategies. It handles automatic routing, actor-critic loops, and multi-agent consensus.

### [💾 Memory Management](./memory/context.md)
The evolutionary substrate. It manages session persistence, knowledge injection (Golden Patterns), and the "Immunity Wall" (Anti-Patterns).

## Scope
- **Strategy Execution**: Implementing the 25+ atomic thinking strategies.
- **Session Lifecycle**: Coordinating the start, step-through, and finalization of cognitive runs.
- **Knowledge Recall**: Dynamic injection of historical context into active reasoning.

## C&C Framework Alignment
The Trinity of Engines provides the mechanics for **Creative** expansion and **Critical** oversight. Sequential logic ensures focus, Fusion logic ensures complexity, and Memory logic ensures that every mission builds upon the system's collective intelligence.
