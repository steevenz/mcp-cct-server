# Domain: Primitives

## Purpose
The Primitives domain defines the **Fundamental Building Blocks** of cognitive expansion. It contains the 22+ unique thinking modes that provide the raw "Creative" material for any mission. These primitives are atomic reasoning strategies that can be combined by the orchestrator to form complex pipelines.

## Scope
Included in this domain:
- **Thinking Engines**: The specific execution logic for modes like `ANALYTICAL`, `CREATIVE`, `SYSTEMATIC`, and `ABDUCTIVE`.
- **Strategy Registry**: Mapping each primitive to its specific meta-prompting template.

Excluded from this domain:
- **Pipeline Orchestration**: Managing the *sequence* of primitives (handled by `Orchestration`).
- **Hybrid Modes**: Complex strategies like Actor-Critic (handled by `Orchestration/Hybrids`).

## Usage
Reference this domain when adding a new atomic thinking mode or refining the prompting strategy for an existing primitive. It provides the "Vocabulary" of thought used by the entire server.

## C&C Framework Alignment
The Primitives domain is the primary engine for the **Creative** pillar of the framework. Each primitive represents a different angle of **Divergent Thinking**, allowing the system to explore multiple paradigms before the Analysis engine applies critical convergence.
