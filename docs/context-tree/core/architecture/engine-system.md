# Engine System: Core Reasoning Modules

The CCT Framework v5.0 architecture is built upon specialized engines that coordinate through a centralized registry.

## 1. Sequential Engine (Logical Depth)
**Purpose**: Enforces linear integrity and deep hierarchical branching.
- **Sequence Guards**: Prevents skipping steps or logical non-sequiturs.
- **Budgeting**: Manages dynamic expansion of thinking steps based on problem complexity.

## 2. Fusion Engine (Synthesis)
**Purpose**: Converges diverse thoughts into a single, high-fidelity conclusion.
- **Convergent Synthesis**: Combines the best elements of multiple reasoning paths.
- **Conflict Resolution**: Identifies and resolves contradictions between actor and critic inputs.

## 3. Memory Engine (Intelligence Persistence)
**Purpose**: Manages long-term and short-term cognitive state.
- **SQLite WAL Layer**: High-performance, thread-safe storage.
- **Pattern Archiver**: Automatically identifies "Golden Patterns" for export to the Context Tree.

## 4. Analysis & Scoring Engine (The Regulator)
**Purpose**: Provides objective metrics for every thought step.
- **Heuristics**: Clarity, Coherence, Evidence, and Novelty scores.
- **Thresholding**: Triggers early convergence if confidence exceeds 0.95 or conventional pivots if quality drops below 0.4.

## Data Flow: The 7-Phase SOP

1.  **Phase 0: Triage**: Identify mission parameters and anti-patterns.
2.  **Phase 1: Research**: Empirical data gathering and landscape analysis.
3.  **Phase 2: Ideation**: Divergent thinking across multiple primitives.
4.  **Phase 3: Critical Analysis**: Convergent pruning and "Council of Critics" review.
5.  **Phase 4: Synthesis**: Fusion of valid branches into a unified model.
6.  **Phase 5: Self-Correction**: Final logic gate and artifact verification.
7.  **Phase 6: Result**: Final output delivery and pattern archiving.
