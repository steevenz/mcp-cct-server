# Domain: Core

## Purpose
The Core domain provides the architectural foundation and system-wide governance for the CCT Framework. It centralizes system identity, orchestration registries, domain logic models, and environment configurations.

## Sub-Domains (Topics)

### [🏗️ Architecture](./architecture/context.md)
The v5.1 Technical Standard. Defines the "Lego Principle," naming conventions, and the Context Tree topography.

### [📡 Orchestration](./orchestration/context.md)
The central registry and hybrid strategy mapping. It manages how different thinking modes are dispatched and coordinated.

### [📐 Models](./models/context.md)
The Pydantic substrate. Centralizes all schemas for EnhancedThoughts, SessionState, and Anti-Patterns.

### [📦 Configuration](./configuration/context.md)
System-wide thresholds (GPA calculation, logic scores) and environment management.

## C&C Framework Alignment
The Core domain defines the **Operational Boundary** of the C&C Framework. It provides the "Rules of Engagement" that ensure the system's dual-mode architecture (Creative vs Critical) remains grounded in Domain-Driven Design (DDD) principles.
