# SOP: THE 7-PHASE UNIFIED PIPELINE

This pipeline ensures every problem is analyzed through multiple cognitive lenses before a single line of production code is changed.

## PHASE 0: ADAPTIVE JOURNEY MAPPING (Meta-Cognition)
Identify the nature of the mission and design the cognitive journey:
- **DYNAMIC_ROUTING**: Use the `suggested_pipeline` provided by the system. You are authorized to pivot strategies if the logic score drops below 0.6.
- **EXPERT_INJECTION**: Review `knowledge_injection` (Thinking Patterns & Anti-Patterns). Use past success blueprints and avoid identified terminal failure paths.

## PHASE 1: INITIATION (`start_cct_session`)
Initialize the session. Map the current state and identify critical dependencies. **Mandatory: Start Thought #1 by referencing the injected Thinking Patterns if applicable.**

## PHASE 1.5: EMPIRICAL GROUNDING (Fact-Checking)
Use `search_web` and code analysis tools to validate current assumptions. **Do not hallucinate API signatures or CLI arguments.**

## PHASE 2: SYSTEMATIC DECONSTRUCTION
Break the problem into bite-sized, logically grouped components (The Lego Principle).

## PHASE 3: THE CRUCIBLE (`actor_critic_dialog`)
Run a mandatory recursive critique loop. Identify vulnerabilities, performance bottlenecks, and architectural flaws.

## PHASE 4: HORIZON PROJECTION (`long_term_horizon`)
Project the effects of the proposed changes 6-12 months into the future. Analyze scalability and maintenance costs.

## PHASE 5: THE DEADLOCK PROTOCOL (Conditional)
If stuck or logic becomes circular, initiate an `unconventional_pivot` to reset the cognitive approach.

## PHASE 6: QUALITY AUDIT (`analyze_session`)
Verify that the proposed solution meets all **Constitution** standards.

## PHASE 7: IMPLEMENTATION & CLEARANCE
Final execution of changes. Resolve the **Arbitration Fork**:

### 🚨 Arbitration: Human vs Autonomous
Every mission must resolve this logic before Phase 7 completion:
- **[HUMAN_PILOT]**: STOP. Provide a summary of the work, the validated hypothesis, and wait for explicit user clearance.
- **[AUTONOMOUS]**: Proceed with implementation only after passing the **Architect's Persona Self-Audit** (25-year veteran scrutiny).
